### 2. **Nueva Tabla: AfectacionValoresParticipeCarga (AVPC)**

**Propósito:**
Registrar **afectaciones manuales** de valores cuando hay novedades que impiden el procesamiento automático.

**Flujo de uso:**
1. Sistema detecta novedad (préstamo no encontrado, múltiples préstamos, etc.)
2. Frontend muestra la novedad al usuario
3. Usuario indica manualmente: préstamo, cuota y valores a afectar
4. Sistema guarda en tabla AVPC
5. Al reprocesar, usa esta tabla como referencia

**Archivos creados:**
- ✅ `CREATE_TABLE_AVPC_AfectacionValoresParticipeCarga.sql` (formato Oracle correcto)
- ✅ `AfectacionValoresParticipeCarga.java` (modelo JPA)
- ✅ `AfectacionValoresParticipeCargaDaoService.java` (interface)
- ✅ `AfectacionValoresParticipeCargaDaoServiceImpl.java` (implementación)
- ✅ `AfectacionValoresParticipeCargaService.java` (interface)
- ✅ `AfectacionValoresParticipeCargaServiceImpl.java` (implementación)
- ✅ `AfectacionValoresParticipeCargaRest.java` (REST API)
- ✅ Constante agregada en `NombreEntidadesCredito.java`

**Estructura de la tabla:**
```sql
CREATE TABLE CRD.AVPC (
    AVPCCDGO NUMBER GENERATED BY DEFAULT AS IDENTITY (START WITH 1 INCREMENT BY 1) NOT NULL,
    NVPCCDGO NUMBER NOT NULL,  -- FK a NovedadParticipeCarga
    PRSTCDGO NUMBER NOT NULL,  -- FK a Prestamo
    DTPRCDGO NUMBER NOT NULL,  -- FK a DetallePrestamo
    -- Valores originales de la cuota
    AVPCVLCT NUMBER(18,2),     -- Valor total cuota
    AVPCVLCP NUMBER(18,2),     -- Capital cuota
    AVPCVLIN NUMBER(18,2),     -- Interés cuota
    AVPCVLDG NUMBER(18,2),     -- Desgravamen cuota
    -- Valores a afectar (usuario)
    AVPCVAFA NUMBER(18,2) NOT NULL,  -- Valor total a afectar
    AVPCCPAF NUMBER(18,2),           -- Capital a afectar
    AVPCINAF NUMBER(18,2),           -- Interés a afectar
    AVPCDGAF NUMBER(18,2),           -- Desgravamen a afectar
    -- Diferencias (calculadas)
    AVPCDFTL NUMBER(18,2),     -- Diferencia total
    AVPCDFCP NUMBER(18,2),     -- Diferencia capital
    AVPCDFIN NUMBER(18,2),     -- Diferencia interés
    AVPCDFDG NUMBER(18,2),     -- Diferencia desgravamen
    -- Auditoría
    AVPCFCAF DATE,             -- Fecha afectación
    AVPCUSAR VARCHAR2(50),     -- Usuario
    AVPCFCRG TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    AVPCOBSR VARCHAR2(1000),   -- Observaciones
    AVPCESTD NUMBER DEFAULT 1, -- Estado
    -- Constraints
    CONSTRAINT PK_AVPC PRIMARY KEY (AVPCCDGO),
    CONSTRAINT FK_AVPC_NVPC FOREIGN KEY (NVPCCDGO) REFERENCES CRD.NVPC(NVPCCDGO),
    CONSTRAINT FK_AVPC_PRST FOREIGN KEY (PRSTCDGO) REFERENCES CRD.PRST(PRSTCDGO),
    CONSTRAINT FK_AVPC_DTPR FOREIGN KEY (DTPRCDGO) REFERENCES CRD.DTPR(DTPRCDGO)
);
```

```java
@Stateless
public class AfectacionValoresParticipeCargaDaoServiceImpl extends EntityDaoImpl<AfectacionValoresParticipeCarga>
```

**Problema:**

- ❌ Recibía `rolPetroComercial` como `Long` (debería ser `String`)
- ❌ Faltaba el estado **8 (DE_PLAZO_VENCIDO)** en la búsqueda
- ❌ Resultado: Todos los registros se enviaban como novedades

**Corrección en `PrestamoDaoService.java`:**
```java
List<Prestamo> selectByEntidadYProductoActivos(String rolPetroComercial, String codigoPetroProducto) throws Throwable;
```

**Corrección en `PrestamoDaoServiceImpl.java`:**
```java
@Override
@SuppressWarnings("unchecked")
public List<Prestamo> selectByEntidadYProductoActivos(String rolPetroComercial, String codigoPetroProducto) throws Throwable {
    String jpql = "SELECT p FROM Prestamo p " +
                 "WHERE p.entidad.rolPetroComercial = :rolPetro " +
                 "AND p.producto.codigoPetro = :codigoProducto " +
                 "AND p.estadoPrestamo IN (2, 8, 10, 11)"; // VIGENTE(2), DE_PLAZO_VENCIDO(8), VIGENTE_POR_REVISAR(10), EN_MORA(11)
    
    Query query = em.createQuery(jpql);
    query.setParameter("rolPetro", rolPetroComercial);
    query.setParameter("codigoProducto", codigoProducto);
    
    List<Prestamo> resultados = query.getResultList();
    return resultados;
}
```

**Corrección en `registrarNovedad()` método:**
```java
private void registrarNovedad(ParticipeXCargaArchivo participe, int tipoNovedad, String descripcion, 
                              Long codigoProducto, Long codigoPrestamo, Double montoEsperado, Double montoRecibido) {

            System.err.println("  ❌ No se puede registrar novedad: ParticipeXCargaArchivo es null");

        if (participe.getCodigo() == null) {

        System.out.println("  → Registrando novedad para partícipe código: " + participe.getCodigo());
```

## 📚 ESTÁNDARES Y BUENAS PRÁCTICAS

### 1. **Estándares para Creación de Tablas Oracle**

**Documento:** `docs/ESTANDARES-CREACION-TABLAS-ORACLE.md`

**Tipos de datos correctos:**
| Tipo Lógico | ❌ Incorrecto | ✅ Correcto |
|-------------|---------------|-------------|
| ID/Código | BIGINT | NUMBER |
| Decimal | DECIMAL(18,2) | NUMBER(18,2) |
| Texto | VARCHAR(n) | VARCHAR2(n) |
| Fecha | DATE | DATE |
| Fecha/Hora | TIMESTAMP | TIMESTAMP |

**PK Autoincremental:**
```sql
-- ✅ Correcto
CODIGO NUMBER GENERATED BY DEFAULT AS IDENTITY (START WITH 1 INCREMENT BY 1) NOT NULL

-- ❌ Incorrecto
CODIGO BIGINT NOT NULL GENERATED ALWAYS AS IDENTITY (START WITH 1, INCREMENT BY 1)
```

**Reglas:**
- Usar `GENERATED BY DEFAULT` (no `ALWAYS`)
- Sin coma entre `START WITH 1` e `INCREMENT BY 1`
- `NUMBER` sin parámetros para IDs

### 2. **Patrón de Implementación de DAOs**

**Regla:** En backend, SIEMPRE crear queries explícitas en el DAO.

**Ejemplo correcto:**
```java
@Stateless
public class MiDaoServiceImpl extends EntityDaoImpl<MiEntidad> implements MiDaoService {

    @PersistenceContext
    EntityManager em;

    @Override
    @SuppressWarnings("unchecked")
    public List<MiEntidad> selectByAlgunCriterio(Long criterio) throws Throwable {
        System.out.println("Ingresa al metodo selectByAlgunCriterio con criterio: " + criterio);
        Query query = em.createQuery(
            " SELECT e " +
            " FROM   MiEntidad e " +
            " WHERE  e.campo = :criterio " +
            " ORDER BY e.codigo "
        );
        query.setParameter("criterio", criterio);
        return query.getResultList();
    }
}
```

**NO usar:**
- ❌ `selectByCriteria()` en el backend (es solo para frontend)
- ❌ Construir `DatosBusqueda` manualmente en el ServiceImpl

### 3. **Nomenclatura de Tablas y Campos**

**Estructura:**
- **Tabla:** 4 caracteres (ej: `AVPC`, `PRST`, `NVPC`)
- **Campo:** `[TABLA][DESCRIPCION]` (ej: `AVPCVLCT` = AVPC + Valor Cuota Total)

**Campos estándar:**
- `[TABLA]CDGO` - Código (PK)
- `[TABLA]ESTD` - Estado
- `[TABLA]FCRG` - Fecha Creación Registro
- `[TABLA]USAR` - Usuario Registro
- `[TABLA]OBSR` - Observaciones
