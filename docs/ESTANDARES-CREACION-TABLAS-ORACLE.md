## 📋 FORMATO ESTÁNDAR PARA CREAR TABLAS

### 1. TIPOS DE DATOS ORACLE

| Tipo Lógico | Tipo Oracle | Ejemplo |
|-------------|-------------|---------|
| ID/Código (PK) | `NUMBER` | `AVPCCDGO NUMBER` |
| Entero largo | `NUMBER` | `AVPCESTD NUMBER` |
| Decimal/Monetario | `NUMBER(18,2)` | `AVPCVLCT NUMBER(18,2)` |
| Texto corto | `VARCHAR2(n)` | `AVPCUSAR VARCHAR2(50)` |
| Texto largo | `VARCHAR2(n)` | `AVPCOBSR VARCHAR2(1000)` |
| Fecha | `DATE` | `AVPCFCAF DATE` |
| Fecha con hora | `TIMESTAMP` | `AVPCFCRG TIMESTAMP` |

**❌ NO USAR:** `BIGINT`, `DECIMAL`, `VARCHAR` (sin el 2)

---

## 2. PRIMARY KEY AUTOINCREMENTAL

### ✅ Sintaxis Correcta (Oracle 12c+):
```sql
AVPCCDGO NUMBER GENERATED BY DEFAULT AS IDENTITY (START WITH 1 INCREMENT BY 1) NOT NULL
```

### ❌ Sintaxis Incorrecta:
```sql
AVPCCDGO BIGINT NOT NULL GENERATED ALWAYS AS IDENTITY (START WITH 1, INCREMENT BY 1)
```

**Notas:**
- Usar `GENERATED BY DEFAULT` (no `GENERATED ALWAYS`)
- Sin coma entre `START WITH 1` e `INCREMENT BY 1`
- `NUMBER` sin parámetros para IDs

---

## 3. FOREIGN KEYS

### ✅ Sintaxis Correcta:
```sql
NVPCCDGO NUMBER NOT NULL,
...
CONSTRAINT FK_AVPC_NVPC FOREIGN KEY (NVPCCDGO) REFERENCES CRD.NVPC(NVPCCDGO)
```

---

## 4. VALORES POR DEFECTO

### ✅ Sintaxis Correcta:
```sql
AVPCESTD NUMBER DEFAULT 1,
AVPCFCRG TIMESTAMP DEFAULT CURRENT_TIMESTAMP
```

## 5. ESTRUCTURA COMPLETA DE UN SCRIPT

```sql
-- =====================================================
-- TABLA: [SCHEMA].[CODIGO] ([NombreCompleto])
-- DESCRIPCIÓN: [Descripción detallada]
-- FECHA: YYYY-MM-DD
-- =====================================================

CREATE TABLE [SCHEMA].[CODIGO] (
    -- PK Autoincrementable automática (Oracle 12c+)
    [PK]CDGO NUMBER GENERATED BY DEFAULT AS IDENTITY (START WITH 1 INCREMENT BY 1) NOT NULL,
    
    -- FK a [Tabla Padre]
    [FK]CDGO NUMBER NOT NULL,
    
    -- CAMPOS DE DATOS
    [TABLA][CAMPO] NUMBER(18,2), -- Descripción
    [TABLA][CAMPO] VARCHAR2(50), -- Descripción
    
    -- AUDITORÍA
    [TABLA]FCRG TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    [TABLA]USAR VARCHAR2(50),
    
    -- ESTADO
    [TABLA]ESTD NUMBER DEFAULT 1,
    
    -- CONSTRAINTS
    CONSTRAINT PK_[CODIGO] PRIMARY KEY ([PK]CDGO),
    CONSTRAINT FK_[CODIGO]_[REF] FOREIGN KEY ([FK]CDGO) REFERENCES [SCHEMA].[TABLA]([PK]CDGO)
);

-- Índices para mejorar performance
CREATE INDEX IDX_[CODIGO]_[FK] ON [SCHEMA].[CODIGO]([FK]CDGO);

-- Comentarios
COMMENT ON TABLE [SCHEMA].[CODIGO] IS 'Descripción de la tabla';
COMMENT ON COLUMN [SCHEMA].[CODIGO].[CAMPO] IS 'Descripción del campo';
```

---

## 6. NOMENCLATURA DE CAMPOS

### Estructura de nombres:
- **Tabla:** 4 caracteres máximo (ej: `AVPC`, `PRST`, `NVPC`)
- **Campo:** 4 caracteres que describen el campo
- **Ejemplo:** `AVPCVLCT` = AVPC (tabla) + VLCT (Valor Cuota Total)

### Campos estándar:
- `[TABLA]CDGO` - Código (PK)
- `[TABLA]ESTD` - Estado
- `[TABLA]FCRG` - Fecha Creación Registro
- `[TABLA]USAR` - Usuario Registro
- `[TABLA]OBSR` - Observaciones

---

## 7. ÍNDICES

### Crear índices para:
- Todas las Foreign Keys
- Campos usados frecuentemente en WHERE
- Campos usados en JOIN

```sql
CREATE INDEX IDX_[TABLA]_[CAMPO] ON [SCHEMA].[TABLA]([CAMPO]CDGO);
```

---

## 8. COMENTARIOS

Siempre agregar comentarios descriptivos:

```sql
COMMENT ON TABLE [SCHEMA].[TABLA] IS 'Descripción completa de la tabla';
COMMENT ON COLUMN [SCHEMA].[TABLA].[CAMPO] IS 'Descripción del campo';
```

## ⚠️ VALIDACIÓN ANTES DE EJECUTAR

1. ✅ Usar `NUMBER` en lugar de `BIGINT`
2. ✅ Usar `NUMBER(18,2)` en lugar de `DECIMAL(18,2)`
3. ✅ Usar `VARCHAR2` en lugar de `VARCHAR`
4. ✅ PK con `GENERATED BY DEFAULT AS IDENTITY`
5. ✅ Sin comas entre START WITH e INCREMENT BY
6. ✅ Todos los campos con comentarios
7. ✅ Índices creados para FKs importantes
8. ✅ Esquema especificado (ej: `CRD.AVPC`)
