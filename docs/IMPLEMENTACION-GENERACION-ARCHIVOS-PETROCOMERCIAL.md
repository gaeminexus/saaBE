## RESUMEN EJECUTIVO

Se ha implementado un sistema completo para generar archivos de descuentos mensuales que se envían a Petrocomercial. El sistema:

✅ Genera archivos TXT con formato de 55 caracteres por línea  
✅ Procesa aportes personales (Jubilación + Cesantía)  
✅ Procesa cuotas de préstamos vigentes  
✅ Suma automáticamente cuotas anteriores pendientes  
✅ Registra todo en base de datos para trazabilidad  
✅ Permite consultar historial de generaciones

## MODELO DE DATOS

### **Tabla 1: GNAP - Generación Archivo Petrocomercial (CABECERA)**

```sql
CREATE TABLE CRD.GNAP (
    GNAPCDGO    NUMBER(19,0) GENERATED BY DEFAULT ON NULL AS IDENTITY,
    GNAPMSPE    NUMBER(10,0)    NOT NULL,   -- Mes del periodo (1-12)
    GNAPANPE    NUMBER(10,0)    NOT NULL,   -- Año del periodo
    GNAPFCGN    TIMESTAMP       NOT NULL,   -- Fecha de generación
    GNAPUSGN    VARCHAR2(50)    NOT NULL,   -- Usuario que generó
    GNAPTLRG    NUMBER(19,0),               -- Total de registros
    GNAPMTEN    NUMBER(19,2),               -- Monto total enviado
    GNAPESTD    NUMBER(5,0),                -- Estado (1=GENERADO, 2=ENVIADO, 3=PROCESADO)
    GNAPRTAA    VARCHAR2(500),              -- Ruta del archivo
    GNAPNMAR    VARCHAR2(200),              -- Nombre del archivo
    GNAPFCEN    TIMESTAMP,                  -- Fecha de envío
    GNAPFCPR    TIMESTAMP,                  -- Fecha de procesamiento
    GNAPOBSR    VARCHAR2(4000),             -- Observaciones
    FLLLCDGO    NUMBER(19,0),               -- FK Filial
    GNAPUSIN    VARCHAR2(50),               -- Usuario ingreso
    GNAPFCIN    TIMESTAMP,                  -- Fecha ingreso
    GNAPUSMD    VARCHAR2(50),               -- Usuario modificación
    GNAPFCMD    TIMESTAMP,                  -- Fecha modificación
    
    CONSTRAINT PK_GNAP PRIMARY KEY (GNAPCDGO)
);
```

**Propósito:** Almacena información de cada generación mensual del archivo.

**Estados:**
- `1` = GENERADO (archivo creado)
- `2` = ENVIADO (archivo enviado a Petrocomercial)
- `3` = PROCESADO (respuesta recibida)

### **Tabla 2: DTGA - Detalle Generación Archivo (RESUMEN POR PRODUCTO)**

```sql
CREATE TABLE CRD.DTGA (
    DTGACDGO    NUMBER(19,0) GENERATED BY DEFAULT ON NULL AS IDENTITY,
    GNAPCDGO    NUMBER(19,0)    NOT NULL,   -- FK Cabecera
    DTGACDPT    VARCHAR2(2)     NOT NULL,   -- Código producto (AH, HS, PE, PH, PQ, PP)
    DTGATLRG    NUMBER(19,0),               -- Total de registros
    DTGAMNTO    NUMBER(19,2),               -- Monto total
    DTGADSCR    VARCHAR2(200),              -- Descripción del producto
    DTGAUSIN    VARCHAR2(50),               -- Usuario ingreso
    DTGAFCIN    TIMESTAMP,                  -- Fecha ingreso
    DTGAUSMD    VARCHAR2(50),               -- Usuario modificación
    DTGAFCMD    TIMESTAMP,                  -- Fecha modificación
    
    CONSTRAINT PK_DTGA PRIMARY KEY (DTGACDGO),
    CONSTRAINT FK_DTGA_GNAP FOREIGN KEY (GNAPCDGO) 
        REFERENCES CRD.GNAP(GNAPCDGO) ON DELETE CASCADE,
    CONSTRAINT UK_DTGA_GN_PT UNIQUE (GNAPCDGO, DTGACDPT)
);
```

**Propósito:** Agrupa y totaliza datos por tipo de producto.

**Códigos de Producto:**
- `AH` = Aportes Voluntarios / Ahorro
- `HS` = Hipoteca Secundaria
- `PE` = Préstamo Emergente
- `PH` = Préstamo Hipotecario
- `PQ` = Préstamo Quirografario
- `PP` = Préstamo Personal

## REGLAS DE NEGOCIO

### **1. Filtrado de Aportes (Producto AH)**

**Condiciones:**
```java
// Entidad en estado ACTIVO
h.entidad.idEstado = 10

// HistorialSueldo en estado 99
h.estado = 99

// Rol Petrocomercial definido
h.entidad.rolPetroComercial IS NOT NULL
h.entidad.rolPetroComercial > 0

// Se toma el registro más reciente por entidad
ORDER BY h.entidad.codigo, h.fechaIngreso DESC
```

**Cálculo del monto:**
```java
montoTotal = montoJubilacion + montoCesantia
```

---

### **2. Filtrado de Préstamos (Productos HS, PE, PH, PQ, PP)**

**Condiciones:**
```java
// Cuota que vence en el mes indicado
dp.fechaVencimiento >= inicioMes
dp.fechaVencimiento <= finMes

// Entidad ACTIVA
dp.prestamo.entidad.idEstado = 10

// Préstamo ACTIVO o VENCIDO
dp.prestamo.estadoPrestamo IN (1, 2)

// Cuota NO pagada ni cancelada anticipadamente
dp.estado NOT IN (4, 7)  // 4=PAGADA, 7=CANCELADA_ANTICIPADA

// Producto con código Petrocomercial
dp.prestamo.producto.codigoPetro IS NOT NULL

// Rol Petrocomercial definido
dp.prestamo.entidad.rolPetroComercial IS NOT NULL
dp.prestamo.entidad.rolPetroComercial > 0
```

**Cálculo del monto (REGLA CRÍTICA):**
```java
// Para cada cuota del mes encontrada:
// 1. Buscar TODAS las cuotas anteriores pendientes
SELECT dp FROM DetallePrestamo dp 
WHERE dp.prestamo.codigo = :codigoPrestamo 
AND dp.numeroCuota < :numeroCuotaActual 
AND dp.estado NOT IN (4, 7)  // Solo pendientes

// 2. Sumar cuotas anteriores pendientes
for (cuotaAnterior : cuotasAnteriores) {
    montoTotal += capital + interes + mora + interesVencido
}

// 3. Sumar cuota del mes actual
montoTotal += capital + interes + mora + interesVencido
```

**Ejemplo:**
```
Préstamo con cuotas:
- Cuota #1 (enero)   - PENDIENTE  - $100
- Cuota #2 (febrero) - PENDIENTE  - $100
- Cuota #3 (marzo)   - PAGADA     - $100  ← Se excluye
- Cuota #4 (abril)   - PENDIENTE  - $100
- Cuota #5 (mayo)    - PENDIENTE  - $100  ← Mes a generar

Al generar para MAYO:
montoEnviado = $100 + $100 + $100 + $100 = $400
(Cuotas #1, #2, #4 y #5)
```

## ENDPOINTS REST

### **Base URL:** `/api/generacion-petro`

### **1. POST `/generar`** - Generar archivo de descuentos

**Request Body:**
```json
{
  "mes": 6,
  "anio": 2025,
  "codigoFilial": 1,
  "usuario": "admin"
}
```

**Response (Éxito):**
```json
{
  "success": true,
  "mensaje": "Archivo generado exitosamente",
  "codigoGeneracion": 123,
  "totalRegistros": 3685,
  "totalMonto": 1234567.89,
  "nombreArchivo": "DESCUENTOS ASOPREP JUNIO 2025.txt",
  "rutaArchivo": "C:\\Users\\admin\\archivos_petrocomercial\\DESCUENTOS ASOPREP JUNIO 2025.txt"
}
```

**Response (Error - Ya existe):**
```json
{
  "error": "Ya existe una generación para el periodo 6/2025",
  "codigoGeneracion": 120
}
```

**Proceso interno:**
1. Valida parámetros
2. Verifica si ya existe generación para ese periodo
3. Crea registro GNAP (cabecera)
4. Recopila aportes (AH)
5. Recopila préstamos (HS, PE, PH, PQ, PP)
6. Agrupa por tipo de producto
7. Crea registros DTGA (resumen por producto)
8. Crea registros PDGA (líneas individuales)
9. Crea registros CXPG (detalle de cuotas) ← PENDIENTE
10. Genera archivo TXT físico
11. Actualiza totales en GNAP

## ESTRUCTURA DEL ARCHIVO TXT

### **Formato de línea: 55 caracteres**

```
| Pos  | Long | Campo              | Descripción                    | Ejemplo       |
|------|------|--------------------|--------------------------------|---------------|
| 1-5  | 5    | codigoParticipe    | Justificado a derecha          | " 9689"       |
| 6-9  | 4    | codigoFijo         | Constante "JRNN"               | "JRNN"        |
| 10-17| 8    | rellenoCeros1      | Relleno de ceros               | "00000000"    |
| 18-25| 8    | fechaProceso       | YYYYMMDD                       | "20250630"    |
| 26-38| 13   | montoDescuento     | Monto × 10,000                 | "0000000450300"|
| 39   | 1    | codigoUno          | Constante "1"                  | "1"           |
| 40-53| 14   | rellenoCeros2      | Relleno de ceros               | "00000000000000"|
| 54-55| 2    | tipoProducto       | AH, HS, PE, PH, PQ, PP         | "AH"          |
```

### **Ejemplo de línea completa:**
```
 9689JRNN00000000202506300000000450300100000000000000AH
```

**Conversión de monto:**
```java
// Monto real: $45.03
long montoEntero = Math.round(45.03 * 10000);  // 450300
String montoFormateado = String.format("%013d", montoEntero);  // "0000000450300"
```

**Orden de registros en el archivo:**
1. Todos los registros AH (ordenados por código partícipe)
2. Todos los registros HS (ordenados por código partícipe)
3. Todos los registros PE (ordenados por código partícipe)
4. Todos los registros PH (ordenados por código partícipe)
5. Todos los registros PQ (ordenados por código partícipe)
6. Todos los registros PP (ordenados por código partícipe)

## NOTAS TÉCNICAS

### **Generación de IDs**
Todas las tablas usan `GENERATED BY DEFAULT ON NULL AS IDENTITY`:
```java
@GeneratedValue(strategy = GenerationType.IDENTITY)
```

### **Tipos de datos**
- `NUMBER(19,0)` → `Long`
- `NUMBER(19,2)` → `Double`
- `NUMBER(10,0)` → `Integer`
- `VARCHAR2(n)` → `String`
- `TIMESTAMP` → `LocalDateTime`
- `DATE` → `LocalDate`

### **EntityManager**
Se inyecta directamente en el REST para ejecutar consultas JPQL:
```java
@PersistenceContext
private EntityManager em;
```

### **Métodos DAO**
Los DAOs heredan de `EntityDao` que expone:
- `save(entity, id)` - Crear/actualizar
- `selectById(id, namedQuery)` - Buscar por ID
- `selectAll(namedQuery)` - Listar todos
