# G40 — CreditoG40

> ⚠️ Si se modifica la lógica de este reporte, actualizar este archivo.

**Tabla destino:** `RPR.CG40`
**Servicio:** `GeneracionG40ServiceImpl.java`
**Se genera:** Solo en **junio (6) y diciembre (12)**. Cualquier otro mes → retorna 0, EJRD OK.
**Fuente:** `CRD.IGFN` (InformacionGeneralFondo) — siempre tiene un único registro.

## Lógica
1. Si el mes no es junio ni diciembre → retornar 0, EJRD OK.
2. Obtener el registro IGFN. Si no existe → retornar 0, EJRD OK.
3. Mapear todos los campos de IGFN directamente a CG40.
4. Persistir el registro.

## Campos

| Campo | Valor |
|---|---|
| `provincia` | **FIJO: `"17"` (Pichincha)** |
| `canton` | **FIJO: `"01"`** |
| `tipoFcpc` | **FIJO: `"3"`** |
| `cambioNombre` | **FIJO: `"NO"`** |
| `tipoAdministracion` | `IGFN.tipoAdministracion` (ingresado desde el frontend) |
| `valorAportePersonalCesantia` | `IGFN.valorAportePersonalCesantia` (ingresado desde el frontend) |
| `valorAportePersonalJubilacion` | `IGFN.valorAportePersonalJubilacion` (ingresado desde el frontend) |
| Resto de campos | Directamente de `IGFN.*` |

> ⚠️ `valorAportePersonalCesantia` y `valorAportePersonalJubilacion` **NO se calculan**.
> Son ingresados manualmente por el usuario en la pantalla de Información General del Fondo y guardados en IGFN.

## Estado IGFN
- Cuando el frontend guarda el IGFN, `saveSingle` setea automáticamente `estado=2` (Modificado).
- El campo `estado` de IGFN **no se usa actualmente** en la generación — el G40 siempre se genera en junio/diciembre con los datos vigentes de IGFN en ese momento.
