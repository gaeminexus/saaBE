# G45 — NuevoParticipeG45

> ⚠️ Si se modifica la lógica de este reporte, actualizar este archivo.

**Tabla destino:** `RPR.CG45`
**Servicio:** `GeneracionG45ServiceImpl.java`
**Fuente:** `RPR.CG41` del mismo EJRC

> G41 corre antes que G45 y cambia `idEstado 1→10`, por eso G45 **no puede filtrar por `idEstado=1`** — lee directamente de CG41 del mismo EJRC.

## Lógica
1. Obtener registros de CG41 del EJRC actual. Si no hay → retornar 0, EJRD OK.
2. Por cada registro de CG41 → enriquecer con Entidad, Exter, Direccion, PerfilEconomico y Participe.
3. Si alguna fuente no existe → campo `null`, no bloquea la generación.

## Campos

| Campo | Fuente |
|---|---|
| `tipoIdentificacion` | `CG41.tipoIdentificacion` |
| `identificacion` | `CG41.identificacion` |
| `cargasFamiliares` | `Entidad.cargasFamiliares` |
| `genero` | `Exter.genero` |
| `estadoCivil` | `Exter.estadoCivil` |
| `fechaNacimiento` | `Exter.fechaNacimiento.toLocalDate()` |
| `profesion` | `Exter.profesion` |
| `provincia` | `Exter.provincia` |
| `canton` | `Exter.canton` |
| `parroquia` | `Direccion` (primera) → `parroquia.nombre` |
| `patrimonio` | `PerfilEconomico.patrimonioNeto` |
| `origenIngresos` | `PerfilEconomico.origenOtrosIngresos` |
| `tipoParticipe` | `Participe.tipoParticipante.nombre` |
| `actividadEconomica` | `Participe.ingresoAdicionalActividad` |
