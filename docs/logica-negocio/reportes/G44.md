> Si cambia, revisar también `CJBM.md` ya que usa la misma lógica base.

**Tabla destino:** `RPR.CG44`
**Servicio:** `GeneracionG44ServiceImpl.java`
**Fuente:** `CRD.ENTD` con `idEstado = 30` (jubilados)

## Lógica
1. Obtener todas las entidades con `idEstado = 30`. Si no hay → retornar 0, EJRD OK.
2. Por cada jubilado mapear campos (SELECTs de COUNT y SUM ejecutados en batch al inicio).

## Campos

| Campo | Fuente | Detalle |
|---|---|---|
| `tipoIdentificacion` | `Entidad.tipoIdentificacion.nombre` | dinámico |
| `identificacion` | `Entidad.numeroIdentificacion` | dinámico |
| `imposicionesAcumuladas` | `CRD.APRT` | COUNT de aportes con `tipoAporte.codigo IN (9, 11)` agrupado por entidad |
| `valorPension` | `CRD.VPPC` | `valorPagar` del primer registro para la entidad (null si no existe) |
| `valorNetoRecibir` | `CRD.VPPC` | Mismo valor que `valorPension` |
| `saldoCuenta` | `CRD.APRT` | SUM de `valor` con `tipoAporte.codigo = 23` (0 si no hay) |
| `jubilacionIess` | — | **FIJO: `"S"`** |
| `tipoJubilacion` | — | `null` |
| `fechaJubilacion` | — | `null` |
| `valoresCompensados` | — | `null` |
